# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

MapBundle ships offline map data (country borders, major cities, major waterways) for .NET apps as
NuGet packages. Data comes from **OpenStreetMap** (ODbL) and is stored as **FlatGeobuf**.
Reading/writing geo formats is delegated to the **GeoConvert** NuGet (`GeoConverter.Read/Write`); this
repo does not re-implement format codecs. GeoConvert cannot read `.osm.pbf` and has no simplify/
reproject/topology ops, so the builder consumes OSM data that already ships in GeoConvert-readable,
pre-generalized form, and uses NetTopologySuite (build-only) for simplify/reproject/coastline.

Two projects:

- **`MapBundle`** (`src/MapBundle/`) — the shipped core library. Multi-targets `net8.0;net9.0;net10.0`.
  Public API: `Maps.Open()` → `MapCatalog` → `Map` → layer `FeatureCollection`s. Depends on GeoConvert
  only (a public dependency, since its `FeatureCollection` is returned to callers). NetTopologySuite is
  **not** a dependency of this project.
- **`Tests`** (`src/Tests/`) — TUnit + Verify. Also hosts the **data-package builder** under
  `src/Tests/Builder/` (namespace `MapBundle.Builder`): it downloads the OSM sources (cached), filters/
  simplifies per region, exports `.fgb`, and writes each data `.nupkg` by hand (`NuPkgWriter`). It is not
  shipped, and is run via the gated `PackageGeneration` test (see Commands), not a separate exe.
  NetTopologySuite is referenced here only.

The data packages (`MapBundle.World`, `MapBundle.[Region]`) are **generated by the builder**, not by
SDK pack. Each carries `buildTransitive/<id>.targets` that copies its `.fgb` files into a
`maps/<Region>` folder in the consumer's output; the core reads from `AppContext.BaseDirectory/maps`.

## Commands

Run from the repo root.

- Build (Debug): `dotnet build src/MapBundle.slnx` — warnings are errors.
- Build + pack the core (Release): `dotnet build src --configuration Release` → core nupkg in `nugets/`.
- Run all tests: build, then run the TUnit **executable** directly:
  `src/Tests/bin/Debug/net10.0/Tests`. Do **not** use `dotnet test` (the .NET 10 SDK rejects TUnit's
  test platform under the VSTest path).
- Run one test: `... Tests --treenode-filter "/*/*/MapTests/*"`.
- Coverage: run the test exe with `--coverage --coverage-output-format cobertura --coverage-output
  unit.cobertura.xml --results-directory TestResults`, then gate with
  `pwsh src/coverage-check.ps1 -Report TestResults/unit.cobertura.xml`. The shipped source
  (`src/MapBundle`) is held at **100% line coverage**; everything under `src/Tests` (including
  `Builder/`) is excluded.
- Build the data packages: set `MAPBUNDLE_BUILD=1` and run the gated generator test:
  `MAPBUNDLE_BUILD=1 src/Tests/bin/Debug/net10.0/Tests --treenode-filter "/*/*/PackageGeneration/Generate"`.
  Output `nugets/`, cache `.cache`; downloads are cached and reused. Without the env var the test is a
  no-op, so normal test runs stay fast. The full build is large (continents + ~200 countries → many
  Geofabrik downloads).
- Validate one region end-to-end (the vertical slice) without building everything: set
  `MAPBUNDLE_SLICE` (comma-separated region ids, default `monaco`) and run the gated `Slice` test:
  `MAPBUNDLE_SLICE=monaco src/Tests/bin/Debug/net10.0/Tests --treenode-filter "/*/*/PackageGeneration/Slice"`.

## Verify snapshots

Snapshot tests use Verify. A mismatch writes `*.received.txt` next to `*.verified.txt` and fails. To
accept, review the diff then replace the `.verified.txt` with the `.received.txt`. `.verified.*` are
committed; `.received.*` are gitignored.

## Conventions (via the `ProjectDefaults` NuGet)

- Central Package Management (`src/Directory.Packages.props`); shared settings in
  `src/Directory.Build.props`. `.slnx` solution.
- Strong-name signing against `src/key.snk` (committed). ProjectDefaults **overwrites the repo-root
  `.editorconfig`** and generates `*.DotSettings` on build — change code style there, not by hand.
- CI: `src/appveyor.yml` (Linux) builds, tests, gates coverage, then runs the **full data build** (the
  gated `PackageGeneration` test with `MAPBUNDLE_BUILD=1`). The `.cache` folder is persisted between
  builds; downloads are cached by **Replicant** (`HttpCache`).

## Builder design notes

(All builder code is under `src/Tests/Builder/`.)

- The region tree is driven by **Geofabrik's `index-v1.json`** (`Geofabrik.cs` → `Regions.cs`):
  continents (top-level entries) plus their direct children (countries). Sub-country levels (US states,
  German Bundesländer) are skipped. `World` is synthetic and merges the continents. `Regions.Build` is a
  pure function over a `GeofabrikEntry` list (testable offline); `Regions.Load` downloads then builds.
- Layers (`MapLayer`, 8): Borders, StatesProvinces, Cities, Rivers, Lakes, Coastline, Land, Ocean.
  Sources: **country-levels** (`CountryLevels.cs`) for Borders (iso1) + StatesProvinces (iso2), selected
  per region by ISO code; **Geofabrik** per-region shapefiles (`gis_osm_places`/`waterways`/`water`) for
  Cities/Rivers/Lakes, filtered by `fclass`; **osmdata** (`OsmData.cs`) global land/ocean polygons
  (EPSG:3857), clipped to the region bbox and reprojected to WGS84. Coastline is derived from the land
  outlines. Cities/Rivers/Lakes for continents/World are built by merging member countries; Borders and
  physical layers are computed directly per region.
- `Geo.cs` is the NetTopologySuite bridge: lossless GeoConvert↔NTS conversion, topology-preserving
  simplify, closed-form EPSG:3857→4326 reprojection, and polygon-outline (coastline) extraction.
- Archives are downloaded (cached by Replicant) and extracted by `Archives.cs` (zip + tar.gz), with an
  `.extracted` marker so repeat runs skip re-unpacking.
- Source URLs are pinned: country-levels is pinned to a release tag (`v2.2.0`, `export_high.tgz` — the
  detailed tier; `low`/`medium` collapse small countries like Monaco to a triangle) for reproducibility;
  Geofabrik/osmdata use `-latest`/`complete` URLs revalidated via the HTTP cache.
- Data packages are licensed **ODbL-1.0** (`NuPkgWriter`), attribution "© OpenStreetMap contributors".
